1、let const var 有什么区别？

let和const相同点：
1.let和const声明的变量，只在声明所在的块级作用域{}内有效
2.都不支持重复赋值
3.都存在暂时性死区
4.都不存在变量提升，必须声明后使用
5.let和const声明的变量都不属于window，var声明的变量相当于挂在window上

let和var的区别：
1.在同一个作用域中不能重复声明相同变量
2.无变量提升
3.块级作用域，存在暂时性死区

2、暂时性死区指的是什么？

​ 在代码块内，使用let命令声明变量之前，该变量都是不可用的，凡是在声明之前就使用就会报错，只能在声 明变量之后再使用。

3、const定义的变量是否能够修改？

可以修改，const声明的是一个只读的常量，一旦声明，常量的值就不能改变，对于简单类型的数据（数值、字符串、布尔值），值就保存在变量指向的那个内存地址，因此
等同于常量，但对于复合类型数据（对象和数组），变量指向的内存地址，保存的只是一个指向实际数据的指针，const只能保证这个指针是固定的（即总是指向另一个固定的地址），
至于它指向的数据结构是不是可变的，就完全不能控制了，因此const声明引用类型的对象的话，值是能改动的，声明基本类型的值就不能改动

4、es6与es5判断是否完全相等的方法是什么？

es5比较两个值是否相等，相等运算符（==）严格相等运算符（===）。前者会自动转换数据类型，后者NaN不等于自身，以及+0等于-0
es6中中比较两个值是否相等 Object.is（） 与（===）相似，不同之处+0不等于-0，NaN等于自身

+0 === -0 //true
NaN === NaN // false
Object.is(+0, -0) // false
Object.is(NaN, NaN) // true

6、如何通过assign实现对象拷贝？ 他的拷贝是深拷贝还是浅拷贝？为什么？

浅拷贝， 如果源对象某个属性的值是对象，那么目标对象拷贝得到的是这个对象的引用
const obj1 = {a:{ b:1 }};
const obj2 = Object.assign({},obj1);
obj1.a.b = 2
obj2.a.b //2
1.第一个参数是目标对象，第二个参数是源对象；
2.如果目标对象与源对象有同名属性，或多个源对象有同名属性，则后面的属性会覆盖前面的属性。
const target = { a: { b:’c’, d:’e’ } }
const sourse = { a: { b: ‘hello’ } }
Object.assign(target,sourse)
//{ a: { b:’hello’ } }
3.Object.assign可以用来处理数组，但是会把数组视为对象

19、for of 与 for in 有什么区别

for of 语句在可迭代对象（Array、Map、Set、String、arguments等）上创建一个迭代循环，调用自定义迭代钩子，并为每个不同属性的值执行语句
for in 一般用于遍历对象的可枚举属性，以及对象从构造函数原型中继承的属性。对于每个不同的属性，语句都会被执行
不建议使用for in 遍历数组，因为输出的顺序不是固定的
如果迭代的对象的变量值是null或者undefided，for in 不执行循环体，建议在使用for in 循环之前，先检查该对象的值是不是null或者undefined

20、filter、map有什么区别

map() 会根据提供的函数对指定序列做映射。
第一个参数function以参数序列中的每一个元素调用function函数，返回包含每次function函数返回值的新列表。
filter（） 函数用于过滤序列，过滤掉不符合条件的元素，返回由复合条件元素组成的新列表
该接收两个参数，第一个为函数，第二个为序列，序列的每个元素作为参数传递给函数进行判断，然后返回true或false，最后将返回true的元素放到新列表中

22、reduce 的使用语法， 都能实现哪些功能

reduce（）累加 初始值，当前元素值，当前索引，调用reduce的数组
arr.reduce（callback,[initialValue]）
callback（执行数组中每个值的函数，包含四个参数）
1.previValue（上一次调用回调返回的值，或者是提供的初始值（initialValue））
2.currentValue（数组中当前被处理的元素）
3.index（当前元素在数组中的索引）
4.array （调用reduce的数组）
initialValue（作为第一次调用callback的第一个参数）
var arr = [1,2,3,4]
var sum = arr.reduce(function(prev,cur,index,arr){
console.log(prev,cur,index);
return prev+cur
},0) //0为初始值
console.log(arr,sum)

26、async await和 promise的区别

async/await 是写异步代码的新方式，以前的方法有回调函数和Promise；async/await是基于Promise实现的，它不能用于普通的回调函数。
async/await与Promise一样，是非阻塞的。async/await是的异步代码看起来像同步代码

async/await前面多了一个async关键字，await关键字只能用在async定义的函数内，async函数会隐式的返回一个promise，该promise的resolve值就是函数return的值，
使用async函数可以让代码简洁很多，不需要想promise一样需要写then，不需要匿名函数处理promise的resolve值，也不需要定义多余的data变量，还避免了的嵌套代码
async/await让try/catch可以同时处理同步和异步错误，promise中需要使用.catch，这样错误处理代码更加复杂
